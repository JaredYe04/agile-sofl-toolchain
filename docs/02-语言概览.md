# Agile-SOFL 语言概览

一份 Agile-SOFL **规格（specification）** 由一个或多个 **模块** 组成，模块之间可以构成父子精化关系。每个模块可选地声明常量、类型、变量、不变式，并包含过程和/或函数定义（空模块也合法）。

文件扩展名通常为 `.asfl`。模块之间用分号分隔；整个规格末尾的分号或句点可选。

---

## 1. 顶层结构

```ebnf
specification ::= modules [ "." ]
modules       ::= topModule { ";" module }
topModule     ::= "module" "SYSTEM_" Identifier [ ";" ] moduleBody "end_module"
module        ::= "module" ( "SYSTEM_" Identifier | Identifier ) [ "/" Identifier ]
                  [ ";" ] moduleBody "end_module"
moduleBody    ::= [ constDecls ] [ typeDecls ] [ varDecls ] [ invDecls ] processFunctionSpecs
```

**系统模块**名称前必须带 `SYSTEM_` 前缀。**子模块**写作 `module Child / Parent;`。

---

## 2. 常量段 `const`

```asfl
const
withdraw_limit = 10000;
transfer_limit = 200000;
```

每项为 `标识符 = 常量表达式`（数字、布尔、`nil`、`<tag>`、集合、序列、映射、`mk_Type(...)` 等），项间分号可选。

---

## 3. 类型段 `type`

```asfl
type
Customers = set of nat;
A = seq of Customers;
Color = {<red>, <green>, <blue>};
Pair = nat * int;
IdMap = map nat to string;
```

类型别名 `Name = 类型表达式`；可写 `Local / Parent.TypeName = …` 引用父模块类型。枚举值**必须**用尖括号。详见 [04-类型与表达式.md](./04-类型与表达式.md)。

---

## 4. 变量段 `var`

```asfl
var
salary: nat;
customers: Customers;
ext#hidden: int;
```

模块级 `ext` / `ext#` 表示外部或隐藏变量。过程体内的 `ext` 段（`rd` / `wr`）语法不同，见 [05-FSF-规约.md](./05-FSF-规约.md)。

---

## 5. 不变式段 `inv`

```asfl
inv
forall[c: customers] | c.deposite > 0;
x > 0 and x <> 10;
```

一条或多条谓词，支持量词与 `and` / `or` / `not`。连续标识符可解析为半形式 `informal_text`。

---

## 6. 过程 `process`

### 6.1 完整过程

```asfl
process A (x, y: int) q1: nat, q2: int, q3: int
ext
rd a: int
wr b: nat
FSF :
x > y && q1 > q2 + q3 ||
x = y && q1 > q2 * q3 ||
others && q1 = q2 + q3
decom: Banking_Decom
comment: informal note
end_process
```

- `(x, y: int)`：**输入**参数组（同组共享类型）
- `q1: nat, q2: int, q3: int`：**输出**参数组（括号后）
- `FSF :` 场景规约；`decom:` 分解图；`comment:` 半形式说明
- 过程名亦可用 `Init` 占位

### 6.2 过程别名

```asfl
process Copy equal Other.P
end_process
```

关键字 **`equal`**（非 `=`）表示等价于 `Other.P`。

---

## 7. 函数 `function`

```asfl
function fee (amount: nat): nat
FSF :
amount > 0 && fee = amount div 100
== if amount > 1000 then amount div 50 else amount div 100
end_function
```

签名 `function 名 (参数): 返回类型`；可选 FSF；体为 `== 表达式` 或 `== undefined`；以 `end_function` 结束。

---

## 8. Banking 完整示例走读

以下为 `tests/fixtures/integration/banking.asfl` 全文，可执行 `npx asfl inspect banking.asfl` 验证。

```asfl
module SYSTEM_Banking;
const
withdraw_limit = 10000;
transfer_limit = 200000;
type
Customers = set of nat;
A = seq of Customers;
var
salary: nat;
customers: Customers;
inv
forall[c: customers] | c.deposite > 0;
process A (x, y: int) q1: nat, q2: int, q3: int
ext
rd a: int
wr b: nat
FSF :
x > y && q1 > q2 + q3 ||
x = y && q1 > q2 * q3 ||
others && q1 = q2 + q3
decom: Banking_Decom
comment: informal note
end_process
end_module
```

**模块与数据**：系统名 `Banking`；常量定义取款/转账上限；`Customers` 为 `set of nat`，`A` 为「客户集合的序列」；`salary`、`customers` 为模块状态。

**不变式**：`forall[c: customers] | c.deposite > 0` 要求集合中每个客户 `deposite` 字段为正。

**过程 A**：输入 `x, y: int`；输出 `q1: nat` 与 `q2, q3: int`；`ext` 读 `a`、写 `b`。FSF 三场景：`x>y` → `q1>q2+q3`；`x=y` → `q1>q2*q3`；`others` → `q1=q2+q3`。`decom`/`comment` 表明非底层过程。

```bash
npx asfl check tests/fixtures/integration/banking.asfl
```

预期无语法错误；FSF 分类器可能给出 info 级提示。
