# 模块边界与仓库结构

Agile-SOFL 采用 **npm workspaces monorepo**。共享库与三条产品线通过 **npm 包 semver** 与 **LSP / JSON 契约** 通信，禁止跨产品 **源码** 引用。

## 包布局

| 包 | 路径 | 职责 |
|----|------|------|
| `@agile-sofl/parser` | `packages/parser` | 词法 / 语法 / 语义 / CLI |
| `@agile-sofl/gui` | `packages/gui` | `.guispec` GUI 规格解析、校验、patch |
| `@agile-sofl/aspec` | `packages/aspec` | `.aspec` 非形式规格解析、style 校验、精化、trace、coverage |
| `@agile-sofl/editor-api` | `packages/editor-api` | 文档 DTO、patch、`ProjectIndex` 再导出 |
| `@agile-sofl/language-server` | `packages/language-server` | LSP stdio 服务 |
| `agile-sofl` | `packages/vscode` | VS Code 扩展（TextMate + Hybrid 装饰 + LSP 客户端） |
| `@agile-sofl/studio` | `packages/studio` | Electron + Vue 可视化编辑器空壳 |

## 允许的依赖

```text
aspec       → (yaml, ajv), @agile-sofl/gui; refine 验收可选 parser
gui         → (yaml)
editor-api  → parser
language-server → parser
vscode      → parser（装饰）；bundled language-server（spawn / server/ 复制产物）
studio      → aspec, gui, editor-api, parser, language-server（spawn dist/server.js）
```

**禁止：**

- `vscode` ↔ `studio` 互引 `src/`
- `language-server` ↔ `vscode` 互引 `src/`（VS Code 仅使用 `dist/server.js` 或 `packages/vscode/server/` 复制产物）
- 任一产品模块相对路径 import 兄弟产品源码

## 本地联调

```bash
npm install
npm run build
npm test
npm run check:boundaries
```

Workspace 依赖使用 `file:../parser` 等形式；发版后改为 semver `^`。

## LSP 契约

产品与 CI 统一 spawn：

```text
node node_modules/@agile-sofl/language-server/dist/server.js --stdio
```

构建：`npm run bundle --workspace @agile-sofl/language-server` 产出自包含 `dist/server.js`。VS Code 扩展通过 `packages/vscode/scripts/copy-lsp-server.mjs` 复制到 `server/`。

## 版本与发版

| 包 | 发版方式 |
|----|----------|
| `@agile-sofl/parser` | npm，`v*` tag |
| `@agile-sofl/aspec` | npm，依赖 yaml / ajv |
| `@agile-sofl/editor-api` | npm，依赖 parser `^` |
| `@agile-sofl/language-server` | npm 可选；扩展 / Studio pin 版本 |
| `agile-sofl` | VSIX，`extension-v*` tag |
| `@agile-sofl/studio` | 私有桌面应用，不发 npm |

## 独立拆仓路径

各 package 已具备独立 `package.json` 与构建脚本，可 `npm publish` 后迁出为独立仓库；产品侧仅保留 npm 依赖与 LSP 二进制契约。

## 边界守卫

根目录 `npm run check:boundaries` 扫描 `packages/vscode/src` 与 `packages/studio/src` 的相对 import，断言无跨产品源码引用。
