# 类型与表达式

**类型表达式**用于变量、参数、量词绑定与返回类型；**表达式**用于常量、函数体与 FSF 定义侧。本文说明类型构造、运算符优先级，以及谓词与条件式写法。

---

## 1. 基本类型与枚举

| 关键字 | 含义 |
|--------|------|
| `nat0` | 非负整数 |
| `nat` | 正自然数 |
| `int`, `real`, `char`, `string`, `bool`, `given` | 常规基础类型 |

枚举值用**尖括号**，逗号分隔，包在 `{ }` 内：

```asfl
type Traffic = {<red>, <amber>, <green>};
```

表达式中可直接写 `<red>`。解析器通过 `EnumValue` token 识别，非 C 风格枚举。

---

## 2. 集合、序列与映射

```asfl
type IdSet = set of nat;
type History = seq of int;
type Scores = map string to nat;
```

**集合** — 字面量与推导：

```asfl
const Empty = {}; Range = {1, 2, 3};
Evens = {n | n: nat & n mod 2 = 0};
```

运算：`union`, `inter`, `diff`, `card`, `get` 等。

**序列** — `[1,2,3]`、`[i | i: int & i > 0]` 或字符串 `"hello"`；运算 `len`, `hd`, `tl`, `conc`。

**映射** — `{->}`、`{1 -> 10, 2 -> 20}`、`{k -> k*2 | k: nat & k < 5}`；运算 `dom`, `rng`, `override`, `comp`。

---

## 3. 组合、积与并类型

**组合类型**（命名字段）：

```asfl
type Account = composed of
  id: nat
  balance: int
end;
```

构造 `mk_Account(1, 100)`；更新 `modify(acc, balance -> 200)`；访问 `acc.balance`。

**积类型** `*`：`Pair = nat * int`；更新 `modify(p, 1 -> 10, 2 -> 20)`（位置从 1 起）。

**并类型** `|`：`Value = nat | int`；顶类型 `universal`。

**跨模块**：`type Local / Parent.Base = nat;`

解析器向前看区分 union（见 `|`）与 product（见 `*`）。

---

## 4. 表达式优先级

数值层级（高 → 低）：

| 层级 | 运算符 |
|------|--------|
| 后缀 | `.field`、`(...)` |
| 一元 | `-` |
| 幂 | `**`（右结合） |
| 乘除 | `*`, `/`, `div`, `rem`, `mod` |
| 加减 | `+`, `-` |

```asfl
const a = 1 + 2 * 3;   (* 7 *)
b = 2 ** 3 ** 2;       (* 2 ** 9 *)
```

类型上下文中的 `*` 表示积类型，表达式中为乘法。

---

## 5. 关系与成员谓词

| 关系符 | 含义 |
|--------|------|
| `=`, `<>` | 相等 / 不等 |
| `<`, `<=`, `>`, `>=` | 序关系 |
| `inset`, `notin` | 集合成员 |

```asfl
inv balance >= 0 and status <> <closed>;
forall[x: ids] | x inset active_ids;
```

布尔应用：`bound(x)`, `subset(s1,s2)`, `is_nat(e)` 等。

谓词层用 **`and` / `or`**（关键字）；**`&&` / `||`** 仅用于 FSF 场景。

---

## 6. 量词

```asfl
inv forall[c: customers] | c.balance > 0;
forall[x, y: int] | x > y or exists[z: int] | z = x - y;
```

格式：`forall`/`exists` `[变量 {,变量}: 类型 {, …}]` 可选嵌套量词 `| 谓词`。量词视为形式构造。

---

## 7. `if` / `let` / `case`

**条件**（条件为 predicate，分支为 expression）：

```asfl
function abs (x: int): int
== if x >= 0 then x else -x
end_function
```

**let** — 赋值 `let x = 1, y = 2 in x + y` 或约束 `let n: nat | n > 0 in n * n`。

**case**：

```asfl
function label (c: Traffic): string
== case c of
     <red> -> "stop"; <green> -> "go"
   ; default -> "unknown"
   end_case
end_function
```

分支 `模式 {,模式} -> 表达式`，`;` 分隔，可选 `default -> …`。

---

## 8. 变量访问

- `name` / 输入快照 `~name`
- `expr.field`、`f(a, b)`
- 模块限定 `Parent.TypeName`

```asfl
process P (req: int) ok: bool
FSF :
req > 0 && ok = true
end_process
```

---

## 9. 实践建议

1. 复杂类型先 `type` 别名，避免深层 `*`/`|` 歧义。  
2. FSF **Def** 用关系式或表达式，勿与场景 `&&`/`||` 混淆。  
3. `npx asfl check` 捕获类型错误。

```bash
npx asfl inspect tests/fixtures/integration/banking.asfl
```

FSF 场景详见 [05-FSF-规约.md](./05-FSF-规约.md)。
