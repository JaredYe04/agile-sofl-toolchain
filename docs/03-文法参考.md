# Agile-SOFL 文法参考

本文给出 **当前解析器实现** 的 EBNF 摘要（`src/parser/parser.ts`、`src/lexer/tokens.ts`）。未列出的构造不保证可解析；以仓库测试为准。

---

## 1. 规格与模块

```ebnf
specification     ::= modules [ "." ]
modules           ::= topModule { [ ";" ] module }
topModule         ::= "module" "SYSTEM_" Identifier [ ";" ] moduleBody "end_module"
module            ::= "module" ( "SYSTEM_" Identifier | Identifier )
                      [ "/" Identifier ] [ ";" ] moduleBody "end_module"
moduleBody        ::= [ constDecls ] [ typeDecls ] [ varDecls ] [ invDecls ]
                      processFunctionSpecs

constDecls        ::= "const" { constItem [ ";" ] }
constItem         ::= Identifier "=" constant
typeDecls         ::= "type" { typeItem [ ";" ] }
typeItem          ::= Identifier [ "/" moduleOrFieldAccess ] "=" typeExpr
varDecls          ::= "var" { varItem [ ";" ] }
varItem           ::= variable ":" typeExpr
variable          ::= "ext" Identifier | "ext" "#" Identifier | Identifier
invDecls          ::= "inv" { predicate [ ";" ] }
processFunctionSpecs ::= { processDef } { functionDef }
```

```asfl
module SYSTEM_Root;
type T = nat;
end_module;
module Child / Root;
var x: T;
end_module.
```

## 2. 过程、函数与 FSF

```ebnf
processDef        ::= processFull | processAlias
processFull       ::= "process" ( "Init" | Identifier )
                      "(" [ paramDecls ] ")" [ paramDecls ] processBody "end_process"
processAlias      ::= "process" Identifier "equal" moduleOrFieldAccess "end_process"
processBody       ::= [ "ext" extVars ] [ "FSF" ":" fsfSpec ]
                      [ "decom" ":" Identifier ] [ "comment" ":" text ]
extVar            ::= ( "rd" | "wr" ) Identifier [ ":" typeExpr ]

functionDef       ::= "function" Identifier "(" [ paramDecls ] ")" ":" typeExpr
                      [ fsfSpec ] [ "==" ( expression | "undefined" ) ] "end_function"
paramGroup        ::= Identifier { "," Identifier } ":" typeExpr

fsfSpec           ::= fsfExpression [ "||" "others" "&&" predicate ]
                      | "others" "&&" predicate
fsfExpression     ::= fsfScenario { "||" fsfScenario }
fsfScenario       ::= predicate "&&" predicate
```

```asfl
module SYSTEM_Demo;
process Dup equal Sub.P
end_process
function id (x: nat): nat == x
end_function
end_module
```

```asfl
FSF :
x > 0 && y > 0 ||
others && y = 1
```

## 3. 类型表达式

```ebnf
typeExpr          ::= unionType | moduleOrFieldAccess | otherType
otherType         ::= productType | typeAtomic
unionType         ::= "universal" | typeAtomic { "|" typeAtomic }
productType       ::= typeAtomic "*" typeAtomic { "*" typeAtomic }
typeAtomic        ::= basicType | enumType | setType | seqType | composedType | mapType

basicType         ::= "nat0"|"nat"|"int"|"real"|"char"|"string"|"bool"|"given"
enumType          ::= "{" EnumValue { "," EnumValue } "}"
setType           ::= "set" "of" typeExpr
seqType           ::= "seq" "of" typeExpr
composedType      ::= "composed" "of" fieldList "end"
mapType           ::= "map" typeExpr "to" typeExpr
```

```asfl
type
S = set of nat;  P = nat * int;  U = nat | int;
E = {<on>, <off>};  M = map string to nat;
C = composed of id: nat name: string end;
```

## 4. 谓词与表达式（摘要）

```ebnf
predicate         ::= conjunction { "or" conjunction }
conjunction       ::= atomicPredicate { "and" atomicPredicate }
atomicPredicate   ::= booleanValue | relationalExpr | booleanApply | quantified
                      | text | "(" atomicPredicate ")" | "not" atomicPredicate

quantified        ::= ( "forall" | "exists" ) [ "not" ] "[" bindingList "]"
                      { quantifierList } "|" predicate
bindingGroup      ::= Identifier { "," Identifier } ":" typeExpr

numberExpr        ::= subNumber1 { ( "+" | "-" ) subNumber1 }
subNumber1        ::= subNumber2 { ( "*" | "/" | "div" | "rem" | "mod" ) subNumber2 }
subNumber2        ::= subNumber3 { "**" subNumber3 }

ifExpr            ::= "if" predicate "then" expression "else" expression
letExpr           ::= "let" patternDef "in" expression
caseExpr          ::= "case" expression "of" caseAlternatives [ ";" defaultExpr ] "end_case"
```

- **IsTypePrefix**：`is_nat`、`is_int`、`is_CustomType` 等。  
- **informal text**：连续 `Identifier` 或含 `StringLiteral` 的短语。  
- 集合/序列/映射支持 `{ e | P }`、`{ e | x: T & P }` 推导形式。

## 5. 词法规则

| 类别 | 规则 |
|------|------|
| `Identifier` | `[A-Za-z_][A-Za-z0-9_]*` |
| `IntegerLiteral` | `-?\d+` |
| `RealLiteral` | `-?\d+\.\d+` |
| `StringLiteral` | `"…"`（`\"` 转义） |
| `CharLiteral` | `'…'` |
| `EnumValue` | `<…>` |

**关键字（节选）**：`module`, `end_module`, `process`, `end_process`, `function`, `const`, `type`, `var`, `inv`, `ext`, `FSF`, `decom`, `comment`, `equal`, `others`, `forall`, `exists`, `composed`, `universal`, `undefined`

**多字符运算符**：`&&`, `||`, `==`, `<>`, `<=`, `>=`, `->`, `**`, `...`, `SYSTEM_`, `mk_`

**注释**：仅 `/* … */` 块注释；空白跳过。**全部整词关键字使用 `\b` 词边界**（如 `informal` 不会拆成 `in`，`total` 不会拆成 `to`）。

---

## 6. 当前解析器支持范围

**已实现**：多模块与 `Child / Parent`；FSF（多场景、`others`、informal）；过程双参数列表、`ext` rd/wr、`decom`、`comment`、别名 `equal`；全部类型构造与枚举尖括号；算术/`if`/`let`/`case`/量词；函数 `== undefined`。

**注意**：

- 类型中 `|` 与 `*` 靠向前看区分 union/product；复杂嵌套宜拆类型别名。  
- 词表含 `forevery`/`forsome`，语法主路径用 `forall`/`exists`。  
- 解析成功不保证类型检查通过。
